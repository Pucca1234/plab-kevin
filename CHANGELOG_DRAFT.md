## 1. 문서 목적
- 이 문서는 날짜별 변경 이력을 기록하는 전용 문서다.
- README와 PRD에 섞여 있던 작업 이력을 분리해 문서 역할을 명확히 한다.

## 2. 작성 규칙
### 2.1 기록 대상
- 사용자 체감 기능 추가
- 기능 정책 변경
- 버그 수정
- 운영/배포/스케줄/인증/복구 관련 변경

### 2.2 기록하지 않을 내용
- 사소한 문구 수정
- 의미 없는 리팩터링 메모
- PRD/README에 이미 충분히 설명된 일반 정책 문장

### 2.3 형식 규칙
- 날짜 단위로 기록한다.
- 분류는 아래 네 가지를 기본으로 사용한다.
  - `Added`
  - `Changed`
  - `Fixed`
  - `Ops`
- 한 줄 요약 위주로 쓴다.
- 원인과 맥락이 필요한 경우에만 짧게 덧붙인다.

### 2.4 운영 규칙
- 날짜별 상세 작업 이력은 이 문서가 source of truth다.
- README와 PRD에는 이 문서 내용을 장문으로 다시 복제하지 않는다.
