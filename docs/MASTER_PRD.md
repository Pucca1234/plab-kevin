# 마스터 PRD

## 목적
이 문서는 여러 기능에 공통으로 적용되는 제품 규칙을 정의합니다. 기능별 상세 동작은 `docs/README.md`에 연결된 기능별 PRD에서 관리합니다.

## 제품 범위
- 소셜 매치 운영 및 기획을 위한 내부 분석 대시보드
- 연, 분기, 월, 주, 일 단위 시계열 비교
- 측정단위별 비교와 드릴다운 분석
- AI 기반 설명과 필터 적용 보조

## 핵심 제품 원칙
- 분석 source of truth는 BigQuery입니다.
- Supabase는 인증, 템플릿, 사용자 설정을 담당합니다.
- 동적 필터는 고정 목록이 아니라 현재 검색 컨텍스트를 반영해야 합니다.
- 드릴다운은 분석 맥락을 유지한 채 하위 단계로 내려가야 하며, 루트 상태로 묵시적으로 리셋되면 안 됩니다.
- 미완료 기간은 확정 추세처럼 다루면 안 됩니다.

## 공통 용어
- `periodUnit`: `year`, `quarter`, `month`, `week`, `day`
- `periodRangeValue`: `recent_8`, `all` 같은 기간 범위 preset 값
- `measurementUnit`: 결과 비교 행의 집계 그레인
- `filterSelectionsByUnit`: 필터 축별 선택값 배열
- `drilldownParent`: 엔티티 드릴다운의 현재 부모 컨텍스트
- `periodDrilldownHistory`: 기간 드릴다운의 상위 경로 기록

## 기능 공통 규칙
### 데이터 정합성
- UI가 사용하는 기존 API 응답 shape를 유지해야 합니다.
- 결측값 표시 정책은 기능 내부에서 일관되어야 합니다.
- BigQuery source dataset은 read-only로 다룹니다.

### 검색 규칙
- 검색 조건은 실제 조회 조건과 표시 컬럼 양쪽에 모두 반영되어야 합니다.
- 선택값이 0개인 필터는 결과 없음으로 해석합니다.
- 현재 선택 가능한 값 전체를 고른 필터는 제한 없음으로 해석합니다.
- 자동 재조회는 허용되지만, 실제 사용자 입력 변화에 의해서만 실행되어야 합니다.

### 기간 규칙
- 미래 기간이나 집계 진행 중 기간은 완료된 값으로 간주하지 않습니다.
- `today`가 포함된 기간은 미완료 기간으로 취급합니다.
- 기간 드릴다운은 상단 검색 의도를 덮어쓰는 동작이 아니라 현재 기간을 더 세분화하는 동작입니다.

### UX 규칙
- 엔티티 드릴다운 메뉴는 테이블 overflow 위에서 보여야 합니다.
- 정렬 버튼과 드릴다운 버튼은 분리된 클릭 영역을 가져야 합니다.
- 기본 탭과 템플릿 탭은 데이터 모델과 UI에서 서로 구분되어야 합니다.

## 공통 검증 기준
- `npm run build` 통과
- `git diff --check` 통과
- 변경한 기능과 관련된 실제 UI/API 흐름 확인

## 기능 분해 구조
- 필터 및 검색
- 드릴다운
- 분석 백엔드
- 템플릿 및 사용자 설정
- AI 어시스턴트
- 인증 및 접근
- 내보내기 및 원본 데이터

## 현재 기능-문서 매핑
- 검색 조건 설정, 동적 필터, 지표 선택: `FILTER_AND_SEARCH_PRD.md`
- 엔티티 드릴다운, 기간 드릴다운, 재조회 충돌 방지: `DRILLDOWN_PRD.md`
- BigQuery provider, serving/source query 경로, 운영 시각 기준: `ANALYTICS_BACKEND_PRD.md`
- 템플릿 저장, 기본 탭, 사용자 설정: `TEMPLATES_AND_PREFERENCES_PRD.md`
- AI 요약, AI 채팅, filter action: `AI_ASSISTANT_PRD.md`
- 로그인, OAuth 콜백, canonical URL: `AUTH_AND_ACCESS_PRD.md`
- Excel/Sheets 내보내기와 원본 데이터 추출: `EXPORT_AND_RAW_DATA_PRD.md`

## 현재 정리 과제
- 기능별 PRD 1차 최신화는 완료됐고, 다음 점검 대상은 인증 가능한 환경에서 수정된 내보내기 경로의 실제 UI 회귀 검증입니다.
- AI/chat 및 일부 UI 소스에는 한글 인코딩 깨짐이 남아 있어 별도 정리 전략이 필요합니다.
- 인증 문서는 현재 로그인 흐름을 담고 있지만, 보호된 화면 접근 정책까지는 아직 별도 정리되지 않았습니다.

## 현재 문서 전환 원칙
- 루트 `PRD.md`는 메인 진입 문서로 유지합니다.
- 상세 정책은 이 문서와 기능별 PRD에 먼저 작성합니다.
- 긴 과거 이력은 레거시 문서와 TODO 문서로 분리합니다.
