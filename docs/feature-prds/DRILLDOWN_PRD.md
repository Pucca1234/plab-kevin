# 드릴다운 PRD

## 목적
엔티티 드릴다운과 기간 드릴다운의 동작을 정의하고, 이후 필터 변경이나 재조회 상황에서도 컨텍스트가 어떻게 유지되는지 설명합니다.

## 구현 진입점
- `app/page.tsx`
- `app/components/EntityMetricTable.tsx`
- `app/components/MetricTable.tsx`
- `app/api/drilldown-options/route.ts`
- `app/api/weeks/route.ts`
- `app/api/heatmap/route.ts`

## 이 문서가 책임지는 범위
- 엔티티 드릴다운과 기간 드릴다운의 생성 규칙
- 드릴다운 후보 단위와 기간 후보 계산 방식
- breadcrumb와 복귀 규칙
- 드릴다운 상태를 유지한 채 재조회가 이뤄져야 하는 규칙

## 이 문서가 직접 책임지지 않는 범위
- 상단 검색 영역의 필터 축 계산
- 검색 조건의 기본값과 필터 옵션 로딩
- 지표 선택 패널
- 일반적인 자동 재조회 예약 정책

위 항목은 `FILTER_AND_SEARCH_PRD.md`에서 정의합니다. 다만 드릴다운 문서는 “드릴다운 상태가 재조회에 의해 잃어버려지면 안 된다”는 보존 규칙을 정의합니다.

## 드릴다운 종류
### 엔티티 드릴다운
- 사용자가 엔티티 행 라벨을 클릭합니다.
- UI는 선택 가능한 하위 측정단위 메뉴를 엽니다.
- 메뉴는 `document.body`에 붙는 portal로 렌더링됩니다.
- 선택하면 `measurementUnit`과 `drilldownParent`를 갱신해 다시 조회합니다.

### 기간 드릴다운
- 사용자가 기간 헤더 라벨을 클릭합니다.
- UI는 해당 기간에서 내려갈 수 있는 하위 기간단위 메뉴를 보여줍니다.
- 선택하면 클릭한 기간 범위 안으로 현재 기간 컨텍스트를 좁힙니다.
- 상위 기간 정보는 `periodDrilldownHistory`에 유지한 채, 조회용 `periodUnit`은 하위 단위로 바뀝니다.

## 현재 동작 규칙
### 엔티티 드릴다운
- 후보 단위는 클라이언트의 candidate map에서 시작하고, `/api/drilldown-options`가 실제 데이터 존재 여부로 최종 필터링합니다.
- 하위 단위를 판단할 때 현재 부모 컨텍스트를 함께 반영합니다.
- breadcrumb를 통해 이전 depth로 복귀할 수 있어야 합니다.

### 기간 드릴다운
- 가능한 하위 단위:
  - `year -> quarter/month/week/day`
  - `quarter -> month/week/day`
  - `month -> week/day`
  - `week -> day`
  - `day -> 없음`
- 기간 드릴다운 클릭 영역과 정렬 클릭 영역은 반드시 분리되어야 합니다.
- breadcrumb는 현재 기간 드릴다운 경로를 보여주고, 루트 기간 조회로 복귀할 수 있어야 합니다.

### 재조회 상호작용
- 필터가 다시 로드되더라도 사용자가 이미 드릴다운된 상태라면 결과는 같은 depth를 유지해야 합니다.
- 늦게 끝난 이전 검색이 더 깊은 드릴다운 결과를 덮어쓰면 안 됩니다.
- 드릴다운으로 들어간 뒤의 재조회는 “루트 검색”이 아니라 “현재 depth의 같은 컨텍스트 재실행”으로 취급해야 합니다.
- 엔티티 드릴다운과 기간 드릴다운은 동시에 존재할 수 있으며, 둘 다 최종 조회 컨텍스트에 유지되어야 합니다.

## 관련 API
- `GET /api/drilldown-options`
- `GET /api/weeks?includeStartDate=1`
- `POST /api/heatmap`

## 예외 케이스
- 선택 가능한 하위 드릴다운이 없으면 깨진 메뉴가 아니라 빈 상태 메시지를 보여줘야 합니다.
- `all` 기간범위 상태의 기간 드릴다운도 정확한 하위 기간 목록을 계산해야 합니다.
- 특히 분기 표기처럼 표시 라벨과 내부 값이 다를 수 있는 경우에도 매핑이 안정적이어야 합니다.
- 드릴다운 후 필터 변경이나 기간 변경이 일어나더라도, 명시적으로 루트 복귀를 선택하지 않는 한 현재 breadcrumb와 parent context가 유지되어야 합니다.

## 검증 항목
- 엔티티 클릭 시 드릴다운 메뉴가 테이블과 스크롤바 위에 떠서 보이는지 확인
- 여러 depth로 들어갔다가 breadcrumb로 이전 단계 복귀가 가능한지 확인
- 드릴다운 상태에서 필터를 바꿔도 결과가 같은 depth를 유지하는지 확인
- 주 단위 컬럼을 클릭해 일 단위로 내려갔을 때 해당 주 안의 일자만 보이는지 확인
- 정렬 버튼이 기간 드릴다운과 독립적으로 동작하는지 확인
