# 필터 및 검색 PRD

## 목적
상단 검색 컨트롤, 동적 필터 계산, 결과 재조회 트리거의 동작을 정의합니다.

## 사용자 목표
사용자는 시간 축, 집계 단위, 지표, 필터를 선택하고 그 조건이 정확히 반영된 결과를 안정적으로 받아야 합니다.

## 이 문서가 책임지는 범위
- 상단 검색 조건이 무엇인지 정의합니다.
- 검색 조건이 어떤 상태로 저장되는지 정의합니다.
- 필터 옵션이 어떻게 계산되고 다시 로드되는지 정의합니다.
- 검색 실행과 자동 재조회가 언제 일어나는지 정의합니다.

## 이 문서가 직접 책임지지 않는 범위
- 엔티티 드릴다운 메뉴의 표시 방식과 후보 단위 판단
- 기간 헤더 드릴다운 메뉴와 breadcrumb UI
- 드릴다운 depth 간 복귀 규칙 자체

위 항목은 `DRILLDOWN_PRD.md`에서 정의합니다. 다만 검색 문서는 “드릴다운 상태도 현재 검색 컨텍스트의 일부로 포함된다”는 점까지만 다룹니다.

## 현재 동작
### 검색 영역 구성
- 1행: 기간단위, 기간범위, 기간 필터
- 2행: 측정단위, 측정단위 기반 필터, 액션 버튼
- 지표 선택은 별도 패널에서 수행하며, 활성 지표는 상단 칩으로 표시합니다.

### 2026-05-29 필터 UX 개선 구현
- `NEXT_PUBLIC_FILTER_UX_V2_ENABLED=1`일 때만 새 필터 UX를 활성화합니다.
- 새 동작에서는 각 필터 후보를 화면 순서 기준 앞선 필터 선택값만으로 계산합니다.
- 뒤쪽 필터 후보를 계산한 결과가 앞쪽 필터 후보를 다시 줄이거나 재정렬하면 안 됩니다.
- 사용자가 필터를 처음 열었을 때 본 후보 리스트는 같은 상위 컨텍스트에서 다시 열어도 유지되어야 합니다.
- 필터 반영 중에는 다른 필터 조작, 조회, 드릴다운, 템플릿 클릭을 잠시 막아 상태 경합을 줄입니다.
- 롤백이 필요하면 env flag를 `0`으로 바꿔 기존 batch 기반 동작으로 되돌립니다.

### 지표 선택 패널
- 카테고리와 담당자 필터는 연동되어야 합니다.
- 카테고리를 선택하면 담당자 목록도 해당 카테고리에 존재하는 항목만 남아야 합니다.
- 검색 입력은 지표명, ID, 설명을 대상으로 동작해야 합니다.
- `선택완료`는 1개 이상 선택 시에만 활성화되어야 합니다.

### 기간 preset
- `year`: `all`, `recent_3`, `recent_5`
- `quarter`: `all`, `recent_4`, `recent_8`, `recent_12`
- `month`: `all`, `recent_6`, `recent_12`, `recent_24`
- `week`: `all`, `recent_8`, `recent_12`, `recent_24`
- `day`: `all`, `recent_7`, `recent_30`, `recent_90`

### 동적 필터 로딩
- 기간 필터 축은 현재 `periodUnit`에 따라 달라집니다.
- 엔티티 필터 축은 현재 `measurementUnit`, 부모 컨텍스트, 데이터 존재 여부, 기간 범위를 기준으로 달라집니다.
- 여러 필터 옵션은 `/api/filter-options-batch`로 한 번에 계산합니다.
- 특정 필터 축의 후보를 계산할 때는 그 필터 축 자신의 active condition을 제외합니다.

### 선택 의미
- 최초 로드 시 각 필터 축은 사용 가능한 전체 값을 선택한 상태로 시작합니다.
- 선택값이 비어 있으면 결과 없음으로 처리합니다.
- 현재 가능한 값을 전부 선택한 상태는 제한 없음으로 처리하며, 실제 좁힘 조건으로 보내지 않습니다.
- 다른 필터 때문에 후보값이 줄어들어도 선택 상태를 임의로 초기화하지 않고 값 기준으로 유지합니다.
- 다중 선택 라벨은 전체 선택, 일부 선택, 0개 선택 상태를 구분해 표시합니다.
- 기간 필터 값은 최신값 우선 정렬을 유지해야 합니다.

### 조회 실행
- 조회는 다음 조건을 함께 사용해야 합니다.
  - 현재 필터에서 계산된 표시 대상 기간
  - 선택한 지표
  - 엔티티/기간 필터의 교집합
  - 존재할 경우 현재 드릴다운 컨텍스트
- 자동 재조회는 필터 로딩 완료 자체가 아니라 실제 사용자 입력 변화에 의해 실행되어야 합니다.

### 검색 상태와 드릴다운 상태의 관계
- `drilldownParent`와 `periodDrilldownHistory`는 검색 실행 시 함께 전달되는 조회 컨텍스트입니다.
- 그러나 이 둘의 생성, 변경, breadcrumb 복귀 규칙은 드릴다운 기능의 책임입니다.
- 검색 기능의 책임은 “현재 드릴다운 컨텍스트를 조회 조건에 포함한다”는 점까지입니다.

## 관련 API
### `GET /api/weeks`
- 기간 목록과 기간 메타데이터 로딩에 사용

### `GET /api/filter-units`
- 현재 측정단위에서 사용할 수 있는 엔티티 필터 축 탐색

### `GET /api/period-filter-units`
- 현재 기간단위에서 사용할 수 있는 기간 필터 축 탐색

### `GET /api/filter-options-batch`
- 여러 필터 축의 옵션 목록을 한 번에 계산

## 예외 케이스
- `measureUnit=all`이어도 기간 필터는 정상 동작해야 합니다.
- source-only 필터 축이 포함되면 backend provider는 source-query 경로를 사용해야 합니다.
- 자동 재조회가 더 최신 검색 결과를 덮어쓰면 안 됩니다.
- 드릴다운 상태에서 필터 변경이 일어나도 현재 depth를 유지해야 합니다.
- 드롭다운 내부 임시 상태와 실제 적용 상태가 어긋나지 않도록 0개 선택 처리 규칙을 유지해야 합니다.

### 자동 재조회 규칙
- `autoSearchPending`은 실제 사용자 입력이 있었을 때만 설정되어야 합니다.
- `isLoadingFilter=false`로 끝났다는 사실만으로는 자동 재조회가 다시 예약되면 안 됩니다.
- 이미 더 최신의 검색 또는 드릴다운 검색이 실행된 경우, 이전 자동 재조회가 화면을 덮어쓰면 안 됩니다.

## 검증 항목
- `periodUnit` 변경 시 해당 축에 맞는 기간 필터들만 노출되는지 확인
- source-only 또는 확장 측정단위 선택 시 필터 옵션이 정상 로드되는지 확인
- 한 필터를 선택했을 때 다른 필터 후보는 좁혀지되 자기 자신의 선택이 초기화되지 않는지 확인
- 0개 선택 상태에서 결과가 빈 상태로 처리되는지 확인
- 결과가 이미 표시된 후 필터를 바꾸면 자동 재조회가 정확히 1회만 실행되는지 확인
