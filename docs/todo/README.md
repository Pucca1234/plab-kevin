# TODO 운영 규칙

## 목적
작업 연속성을 채팅이 아니라 저장소 안의 문서로 남겨, 다음 작업자가 바로 이어서 시작할 수 있게 합니다.

## 파일 구성
- `ACTIVE_TODO.md`: 지금 처리해야 할 작업과 진행 중 작업
- `BACKLOG_TODO.md`: 바로 처리하지 않을 작업과 구조 개선 항목

## 운영 규칙
- 큰 작업을 시작하기 전에는 `ACTIVE_TODO.md`를 먼저 확인합니다.
- 작업 중 새 의존성, 리스크, 후속 과제가 생기면 바로 TODO를 추가하거나 갱신합니다.
- 작업이 끝나면 완료 항목을 표시하고, 거기서 파생된 후속 작업을 남깁니다.
- 즉시 처리하지 않을 항목은 기능 PRD에 애매하게 남기지 말고 `BACKLOG_TODO.md`로 옮깁니다.

## 항목 형식
- ID
- 상태
- 발생 출처
- 중요 이유
- 다음 액션
- 참고 문서 또는 파일

## 상태 의미
- `todo`: 바로 잡을 수 있는 대기 작업
- `in_progress`: 현재 진행 중
- `blocked`: 외부 결정이나 의존성 때문에 멈춤
- `done`: 완료됨
